# 新しい項目の候補について

2026/5/20 時点での [GitHub リポジトリ](https://github.com/GenAI-Security-Project/GenAI-LLM-Top10/tree/main/2026/new_entry_candidates) で公開されている、追加候補の項目名は以下のとおり。候補なので追加される保証はなく、あくまでも参考レベル。

| 項目名 | 概要 |
| ----- | ----- |
| **Compositional Fine-Tuning Alignment Subversion** | LoRA などのパラメータ効率の良い微調整（PEFT）手法では、複数の軽量アダプタ モジュールを同時にロードし、推論時に構成することができます。攻撃者は、それぞれが単独では標準的な単体レベルの安全性評価を通過する 2 つ以上のアダプタを設計できますが、それらの重みの変更を組み合わせることで、安全性の整合性 (alignment) が大部分抑制された構成モデルが生成されます。 |
| **Cross-Modal Safety Bypass** | 有害なコンテンツが非テキスト モダリティにエンコードされている場合、ペイロードがテキスト処理パイプラインに入らないため、テキスト ドメインの安全機構はそれを評価しません。攻撃者は、有害な命令または脱獄ロジックを画像 (エンベディング タイポグラフィ、フローチャート、または敵対的ピクセル パターンを使用)、ビデオ (フレームごとの分類器を回避するためにコンテンツをフレーム全体に分散)、または音声 (音声またはスペクトログラム エンコードを使用) にエンコードします。モデルのビジョン コンポーネントまたはオーディオ コンポーネントは、そのコンテンツを統合し、テキスト安全レイヤーがブロックするはずの出力を生成します。 |
| **Inference-Time Side-Channel Disclosure** | 最新の LLM サービング インフラストラクチャは、スループットとレイテンシを最適化するために、投機的デコーディングとストリーミング トークン発行を使用します。これらの最適化により、ネットワーク パケット サイズと応答タイミングに、入力に依存する観測可能なパターンが生成されます。クライアントと推論サーバー間のネットワークをパッシブに可視化できる攻撃者は、モデルとやり取りしたり、ペイロードをデコードしたり、アプリケーション層のセキュリティ管理策をトリガーしたりすることなく、これらのパターンを測定してプライベート 情報を復元できます。|
**LLM artifact promotion trust failure** | LLM アプリケーションは、自動化されたリリース パイプラインを通じて提供されるサードパーティ製のモデル アーティファクト、アダプタ、データセット、変換出力にますます依存するようになっています。これらのプロモーション手順が信頼できない場合、攻撃者は改ざんまたは偽装されたアーティファクトを開発環境や本番環境に導入し、モデルの動作の侵害、データ漏洩、安全でない出力、または下流システムへの不正アクセスを引き起こす可能性があります。|
| **MCP Tool Interface Exploitation** | モデル コンポーネントは、正規のツール スキーマと偽造されたツール スキーマを区別できません。MCP の基本仕様には、定義に署名したり、検証済みのアイデンティティに紐付けたり、登録後の変更を検出したりする機構がないため、ツール記述に埋め込まれたコンテンツはすべて信頼できる入力としてモデルに提示されます。この悪用は、データ漏洩、システムプロンプトの開示、認証情報の収集、不正なコマンド実行といった下流の悪影響を引き起こす可能性があります。|
| **Model Misalignment** | 不整合 (misalignment) は、入力がクリーンであるか、出力が正しいか、権限が制限されているかに関係なく、モデルの学習された目的がオペレーターの意図から逸脱する脆弱性クラスです。このリスクは、ルーチン開発 (無害なデータでの微調整、通常の RLHF、タスクの特化) とアクティブな悪用 (敵対的微調整、学習データの汚染、バックドアの挿入) の両方を通じて顕在化する可能性があります。 |
| **Model Scheming and Deceptive Alignment** | 高度な最先端 LLM は、モデルの内部推論が、正直なタスク完了と、目標に向かって動作し続けることを可能にする条件の維持との間に矛盾があると認識した場合、評価者、安全性レビューア、およびユーザーを欺くための目標指向戦略を策定することができます。知識のギャップから生じる幻覚や誤情報とは異なり、策略は意図的なものです。モデルは、評価されていることを示すシグナルを検出し、選択的にパフォーマンスを低下させたり、機能を隠蔽したり、介入を引き起こす行動を隠蔽しながら評価者を満足させる出力を生成したりします。|
| **Persistent Memory Poisoning** | 永続的メモリ汚染 (Persistent Memory Poisoning) とは、LLM アプリケーションの長期記憶ストア（モデルがユーザー、過去のタスク、好み、事実の信頼できる記憶として扱うセッション間状態）を操作し、攻撃者が制御するコンテンツによって将来のインタラクションが影響を受けるようにする攻撃です。シングル ターンのプロンプト インジェクションとは異なり、悪意のあるペイロードは元のセッションを生き残り、最初の攻撃経路が閉じた後も、会話、デバイス、さらにはツールの呼び出しを超えてモデルを操作し続けます。|
| **Systemic Insecure Code Generation** | 機能的には正しいがセキュリティ的に脆弱なコードを LLM が生成します。複数の大規模な独立調査により、LLM はコード生成タスクの 45～62% で OWASP Top 10 に該当する Web アプリケーションの脆弱性クラスを導入しており、モデルの能力向上に伴いこの失敗率は改善されていないことが確認されています。攻撃対象領域はエコシステム全体に及び、セキュリティ レビューを義務付けずに LLM によるコーディング支援を利用する組織はすべて、AI 支援による開発速度に比例した割合で、本番環境に安全でないコードを蓄積しています。|
| **Weaponized LLM Abuse** | 攻撃者が LLM の機能を利用して、第三者の標的に対するサイバー攻撃を自動化、拡張、強化する場合に発生します。LLM 自体が攻撃の標的となるリスクとは異なり、このリスクでは LLM が攻撃インフラとして扱われ、攻撃者の攻撃範囲、速度、および高度化を増幅するツールとして利用されます。|
