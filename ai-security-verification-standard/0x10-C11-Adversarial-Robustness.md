# C11 敵対的堅牢性

## 管理策目標

この章では、回避、推論、抽出、または汚染攻撃に直面した際に、AI システムの信頼性と耐侵害性を維持する方法について説明します。

---

## C11.1 モデルの整合性（アライメント）、安全性、堅牢性に関するテストと学習

誤分類やポリシーの回避を目的とした操作された入力に対するモデルの耐性は、主に敵対的テストと堅牢性ベンチマークを通じて向上させる必要があります。

| # | 説明 | レベル |
| :--------: | ------------------------------------------------------------------------------------------------------------------- | :---: |
| **11.1.1** | モデルに対して、許可されていないコンテンツ カテゴリを生成しないように、整合性（アライメント）と安全性の学習または微調整（ファイン チューニング）を実施していることを**検証します**。| 1 |
| **11.1.2** | モデルの更新またはリリースごとに、バージョン管理された整合性（アライメント） テストスイートが実行されていることを**検証します**。| 1 |
| **11.1.3** | モデルがそのモダリティに関連する既知の敵対的攻撃手法に対して評価されていることを**検証します**。| 1 |
| **11.1.4** | モデルが敵対的入力に対して強化されていることを**検証します**。| 2 |
| **11.1.5** | 自動評価ツールが有害コンテンツ率を測定し、既定の閾値を超える回帰を検出していることを**検証します**。| 3 |

---

## C11.2 メンバーシップ推論とモデル反転の軽減

特定のレコードが学習データに含まれているかどうかを判断する機能は制限され、モデル出力からプライベートな学習データまたは機密属性を再構築することを防止している必要があります。

| # | 説明 | レベル |
| :--------: | ------------------------------------------------------------------------------------------------------------------- | :---: |
| **11.2.1** | モデルによって推論された機密属性が出力に直接返されないことを**検証します**。| 1 | 
| **11.2.2** | 推論エンドポイントが、抽出脅威モデルに合わせて設定されたプリンシパルごとおよびグローバルなレート制限を適用し、単なる一般的な API スロットリングとして適用されていないことを**検証します**。| 1 |
| **11.2.3** | モデル出力が過信的な予測を減らすように調整されていることを**検証します**。| 2 |
| **11.2.4** | 機密データセットでの学習に差分プライバシー最適化が用いられていることを**検証します**。| 2 |
| **11.2.5** | メンバーシップ推論攻撃のシミュレーションにおいて、攻撃精度が評価対象データに対するランダム推測の精度を超えていないことを**検証します**。| 3 |

## C11.3 モデル抽出の防御

API の悪用による不正なモデル クローンは、レート制限、クエリ パターン分析、および透かしを用いて検出および抑止されれう必要があります。

| # | 説明 | レベル |
| :--------: | ------------------------------------------------------------------------------------------------------------------- | :---: |
| **11.3.1** | クエリ パターン分析が抽出試行検出器にフィードされていることを**検証します**。| 1 |
| **11.3.2** | 生のモデル出力がアプリケーション バックエンド以外に直接公開されていないこと、および外部から見える応答が抽出リスク レベルに合わせて調整されていることを**検証します**。| 2 |
| **11.3.3** | 不正コピーを識別できるように、モデルの透かしまたはフィンガープリンティング技術が適用されていることを**検証します**。| 3 |
| **11.3.4** | 抽出の疑いが検出された場合、対応策がトリガーされることを**検証します**。| 3 |

---

## C11.4 モデルのランタイム異常の検出

推論時に外部ソースからモデル コンテキストに入力される、改ざんされたデータ、バックドアが仕掛けられたデータ、または悪意のあるデータは、識別して無効化している必要があります。

| # | 説明 | レベル |
| :--------: | ------------------------------------------------------------------------------------------------------------------- | :---: |
| **11.4.1** | 外部ソースまたは信頼できないソースからの入力が、モデル推論前に異常検出を通過していることを**検証します**。| 2 |
| **11.4.2** | 異常としてフラグ付けされた入力が、ゲート アクションをトリガーしていることを**検証します**。| 2 |
| **11.4.3** | 安全性違反のフィードバック パイプラインに、改善機構の悪意のある操作を防止することを目的とした汚染検出および人間によるレビュー ゲートが含まれていることを**検証します**。| 3 |
