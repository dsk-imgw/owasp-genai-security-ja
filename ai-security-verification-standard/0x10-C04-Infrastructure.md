# C4 インフラストラクチャ、構成およびデプロイメントのセキュリティ

## 管理策目標

この章では、AI 固有のインフラストラクチャ コンポーネントを、モデルの盗難、データ漏洩、およびテナント間汚染から保護するための対策について説明します。

---

## C4.1 AI ワークロードのサンドボックス化および検証

信頼できない AI モデルはセキュアなサンドボックスに隔離し、機密性の高い AI ワークロードは、トラステッド実行環境（TEE）および機密コンピューティング技術を使用して保護する必要があります。

| # | 説明 | レベル |
| :--------: | ------------------------------------------------------------------ | :---: |
| **4.1.1** | AI モデルが隔離されたサンドボックスで実行されることを**検証します**。 | 1 |
| **4.1.2** | モデル アーティファクトのロード時に、逆シリアル化中に任意のコード実行を許可しないシリアル化形式の明示的な許可リストが適用されていることを**検証します**。 | 1 |
| **4.1.3** | ワークロード アテステーションがモデル ロード前に実行され、実行環境が改ざんされていないことを証明していることを**検証します**。| 3 |
| **4.1.4** | 機密推論サービスが隔離された実行環境を通じて実行時にモデルの重みを保護していることを**検証します**。| 3 |

---

## C4.2 AI ハードウェアのセキュリティ

GPU、TPU、専用 AI アクセラレータなどの AI 専用ハードウェア コンポーネントは、セキュリティが確保されている必要があります。

| # | 説明 | レベル |
| :--------: | ------------------------------------------------------------------------------------------ | :---: |
| **4.2.1** | AI アクセラレータ（GPU） ファームウェアが、バージョン固定、署名、およびブート時のアテステーションを受けていることを**検証します**。| 2 |
| **4.2.2** | トラステッド実行環境（TEE）内での実行が、ハードウェアによる隔離、メモリ暗号化、および完全性保護を提供していることを**検証します**。| 3 |
| **4.2.3** | 各ワークロードの実行前に、ハードウェア ベースのアテステーション機構を使用して AI アクセラレータ（GPU）の完全性が検証されていることを**検証します**。| 3 |
| **4.2.4** | ジョブ間のメモリ無害化を伴うパーティショニング機構によって、アクセラレータ（GPU） メモリがワークロード間で分離されていることを**検証します**。| 3 |
| **4.2.5** | アクセラレータの相互接続が、承認されたトポロジと認証されたエンドポイントに制限されていることを**検証します**。| 3 |

---

## C4.3 エッジおよび分散型 AI のセキュリティ

エッジ コンピューティング、連合学習（フェデレーテッド ラーニング）、マルチ サイト アーキテクチャを含む分散型 AI のデプロイメントは、セキュリティを確保する必要があります。

| # | 説明 | レベル |
| :--------: | ------------------------------------------------------------------------------------------ | :---: |
| **4.3.1** | エッジ AI デバイスが強力な認証機構を使用して中央インフラストラクチャに対して認証を行っていることを**検証します**。| 1 |
| **4.3.2** | エッジ デバイスまたはモバイル デバイスにデプロイされるモデルがパッケージング時に暗号署名され、デバイス上のランタイムがロードまたは推論の前にこれらの署名またはチェックサムが確認 (validate) されていることを**検証します**。| 2 |
| **4.3.3** | 推論ランタイムがプロセス、メモリ、およびファイル　アクセスの分離を強制していることを**検証します**。| 3 |
| **4.3.4** | ローカルに保存されているモデルの重みと機密パラメータが、ハードウェア ベースの鍵ストアまたはセキュア エンクレーブを使用して暗号化されていることを**検証します**。| 3 |
| **4.3.5** | モバイル、IoT、または組み込みアプリケーションにパッケージ化されたモデルが、保存時に暗号化され、信頼できるランタイムまたはセキュア エンクレーブ内でのみ復号され、アプリ パッケージまたはファイル システムからの直接抽出が防止されていることを**検証します**。| 3 |
