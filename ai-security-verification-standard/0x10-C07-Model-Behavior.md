# C7 モデルの動作、出力制御および安全性の保証

## 管理策目標

この章では、安全でない、形式が不正な、またはリスクの高い応答がユーザーや下流システムに到達しないように、モデル出力の制約、検証、および監視について説明します。

---

## C7.1 出力フォーマットの強制

下流システムへのインジェクション リスクを低減するために、モデル出力は構造化され、検証される必要があります。

| # | 説明 | レベル |
| :--------: | --------------------------------------------------------------------------------------------------------------------- | :---: |
| **7.1.1** | アプリケーションがすべてのモデル出力を定義済みのスキーマに対して検証し、一致しない出力を拒否していることを**検証します**。| 1 |
| **7.1.2** | モデル生成の出力が長さ制限と終了制御によって制限されていることを**検証します**。| 1 |

---

## C7.2 幻覚（ハルシネーション）の検出および軽減

潜在的に不正確または捏造されたコンテンツを検出し、信頼性の低い出力がユーザーや下流システムに到達しないようにする必要があります。

| # | 説明 | レベル |
| :--------: | --------------------------------------------------------------------------------------------------------------------- | :---: |
| **7.2.1** | システムが信頼度推定方法を使用して生成された回答の信頼性を評価していることを**検証します**。| 2 |
| **7.2.2** | 信頼度スコアが定義されたしきい値を下回った場合に、アプリケーションが回答を自動的にブロックするか、フォールバック メッセージに切り替えていることを**検証します**。| 2 |
| **7.2.3** | ポリシーによって高リスクと分類された回答に対して、システムが追加の検証ステップを実行していることを**検証します**。| 3 |

---

## C7.3 出力の安全性

技術的管理策によって、危険なコンテンツはユーザーに表示される前に検出され、削除される必要があります。

| # | 説明 | レベル |
| :--------: | --------------------------------------------------------------------------------------------------------------------- | :---: |
| **7.3.1** | 自動分類器がすべての応答をスキャンし、既定の有害コンテンツ カテゴリに一致するコンテンツをブロックしていることを**検証します**。| 1 |
| **7.3.2** | 出力フィルタが、システム プロンプトのコンテンツまたはバックエンド データを開示する応答を検出してブロックしていることを**検証します**。| 2 |
| **7.3.3** | モデル生成の出力がアウトバウンド リクエストをトリガーしていないことを**検証します**。| 2 |
| **7.3.4** | モデル出力が、同形文字、書式設定、メタデータ、または構造化フィールドによって作成された、隠蔽、エンコード、または誤解を招くコンテンツについてチェックされていることを**検証します**。| 3 |

---

## C7.4 参照元帰属と引用の完全性

RAG に基づいた出力は、その引用となる文書まで遡って追跡可能でなければならず、引用された記述 (claim) は、取得されたコンテンツによって検証可能な形で裏付けられている必要があります。

| # | 説明 | レベル |
| :--------: | --------------------------------------------------------------------------------------------------------------------- | :---: |
| **7.4.1** | 検索拡張生成（RAG）を使用して生成された応答に、ソース文書への帰属情報が含まれていることを**検証します**。| 1 |
| **7.4.2** | 出所情報が捏造されることがないように、RAG の帰属情報は検索メタデータから取得され、モデルによって生成されたものではないことを**検証します**。| 1 |
| **7.4.3** | RAG 応答内の記述 (claim) が、取得されたチャンクに追跡可能であることを**検証します**。| 2 |
| **7.4.4** | 生成されたメディアに、AI によって生成されたことを証明する透かしが入っていることを**検証します**。| 3 |
