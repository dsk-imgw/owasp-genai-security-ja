# C6 モデルのサプライチェーン セキュリティ

## 管理策目標

この章では、サードパーティのモデル、フレームワーク、またはデータセットを悪用してバックドア、バイアス、または悪用可能なコードを埋め込む AI サプライチェーン攻撃に対する防御について説明します。

---

## C6.1 モデル アーティファクトの完全性

サードパーティのモデルの出所は、微調整（ファイン チューニング）またはデプロイ前に認証され、隠れた動作がないかチェックされる必要があります。また、AI アーティファクトは承認されたソースからのみダウンロードする必要があります。

| # | 説明 | レベル |
| :--------: | ------------------------------------------------------------------------------------------------------------------- | :---: |
| **6.1.1** | モデルをインポートする前に、悪意のあるコードがないかスキャンされていることを**検証します**。| 1 |
| **6.1.2** | モデルの重み、データセット、および微調整（ファイン チューニング）アダプタが承認されたソースからのみダウンロードされていることを**検証します**。| 1 |
| **6.1.3** | すべてのサードパーティのモデル アーティファクトが完全性検証可能であることを**検証します**。| 2 |
| **6.1.4** | モデルが開発環境以外の環境にデプロイされる前に、動作受け入れテスト スイートに合格していることを**検証します**。| 2 |

---

## C6.2 AI BOMとサプライチェーン監視

サプライチェーンの侵害事象に対応できるよう、AI 固有の詳細な部品表（BOM）を作成し、署名する必要があります。

| # | 説明 | レベル |
| :--------: | ------------------------------------------------------------------------------------------------------------------- | :---: |
| **6.2.1** | すべてのモデル アーティファクトが、データセット、重み、ライセンス、およびデータの出所のステートメントを記載した、バージョン管理され機械可読な AI BOM を公開していることを**検証します**。| 1 |
| **6.2.2** | AI BOM がデプロイメント前に暗号署名されていることを**検証します**。| 2 |
| **6.2.3** | AI BOM の完全性チェックにおいて、コンポーネントのメタデータが欠落している場合、ビルドが失敗することを**検証します**。| 2 |
