# C9 オーケストレーションおよびエージェントのセキュリティ

## 管理策目標

この章では、自律システムおよびマルチ エージェント システムが、承認・意図・制限されたアクションのみを実行することを保証する方法について説明します。

---

## C9.1 実行予算、ループ制御、およびサーキット ブレーカー

ランタイム拡張（再帰、並行処理、コスト）は制限され、暴走動作が発生した場合は安全に停止する必要があります。

| # | 説明 | レベル |
| :--: | --- | :---: |
| **9.1.1** | ツールごとのクォータとタイムアウト（CPU、メモリ、ディスク、出力、実行時間など）が適用されていることを**検証します**。| 1 |
| **9.1.2** | 実行ごとの予算（最大再帰深度、トークン使用量、金銭的支出など）がランタイムによって設定され、適用されていることを**検証します**。| 1 |
| **9.1.3** | すべてのアクティブなエージェント インスタンスを停止できる、スウォーム レベル（マルチ エージェント全体）のキルスイッチが存在していることを**検証します**。 | 2 |

---

## C9.2 高影響アクションの承認と不可逆性の制御

特権アクション、高影響アクション、または不可逆アクションには、信頼できる承認チェックポイントが必要です。

| # | 説明 | レベル |
| :--: | --- | :---: |
| **9.2.1** | エージェント ランタイムが、明示的な人間の承認が受信され検証されるまで、特権アクション、高影響アクション、または不可逆アクションの実行をブロックしていることを**検証します**。 | 1 |
| **9.2.2** | 承認要求が、差分、コマンド、受信者、金額、リソース、スコープなどの正規化された完全なアクション パラメータを、切り捨てや安全でない変換なしに表示していることを**検証します**。 | 2 |
| **9.2.3** | 各高影響アクションに、読み取り専用、自己可逆、外部的に可逆、不可逆などの信頼できる可逆性分類が設定されていることを**検証します**。| 2 |
| **9.2.4** | エージェント ランタイムが、アクションの影響と可逆性に基づいて、アクションをブロック、承認要求、または制限することにより、可逆性分類を強制していることを**検証します**。| 2 |
| **9.2.5** | 自己変更機能（プロンプトの書き換え、ツール リストの変更、パラメータの更新など）が、強制可能な境界によって制限されていることを**検証します**。| 2 |
| **9.2.6** | エージェント型システムには、実行前に計画された高リスクな行動について、AI を活用したレビューが含まれていることを**検証します**。このレビューは、決定論的なポリシー ゲートを補完するものであり、それに取って代わるものではありません。| 2 |
| **9.2.7** | AI を活用したレビュー機構が、悪意のある入力による操作から保護されており、プロンプト インジェクションによってオーバーライドまたは迂回されないことを**検証します**。| 2 |
| **9.2.8** | 承認が、アクション パラメータ、要求者のアイデンティティ、実行コンテキスト、および一意の単回使用ノンスと暗号的に紐付けられていることを**検証します**。| 3 |
| **9.2.9** | 承認の発行に使用される暗号鍵マテリアルまたは資格情報が、エージェント ランタイムから分離されていることを**検証します**。| 3 |
| **9.2.10** | 多段階または複数エージェントによるアクションチェーンの承認ゲートは、そのチェーン内のどこかに存在する影響度が最も高い可逆性分類を強制していることを**検証します**。| 3 |

## C9.3 コンポーネントの分離とツールの認可

ツールとプラグインの実行、ロード、および出力は、不正なシステム アクセスと安全でない副作用を防止するために制限される必要があります。

| # | 説明 | レベル |
| :--: | --- | :---: |
| **9.3.1** | 各ツール/プラグインが最小権限サンドボックス内で実行されているか、モデル操作から隔離されていることを**検証します**。| 1 |
| **9.3.2** | ツールの出力がスキーマに対して確認 (validate) されていることを**検証します**。| 1 |
| **9.3.3** | ツール マニフェストに必要な権限、リソース制限、および出力の確認 (validation) 要件が宣言されていることを**検証します**。| 2 |
| **9.3.4** | ランタイムがツール マニフェストで宣言された権限、リソース制限、および出力の確認 (validation) 要件を強制していることを**検証します*。| 2 |
| **9.3.5** | 信頼できないデータを処理するコンポーネントがツール呼び出し機能から隔離され、侵害されたデータの処理によって不正なツール呼び出しがトリガーされていないことを**検証します**。| 2 |
| **9.3.6** | 信頼できないツール出力の処理とエージェント操作の間にアーキテクチャ上の分離があることを**検証します**。| 2 |
| **9.3.7** | モデル出力で指定された外部リソースが、エージェントがインストールまたは呼び出す前に、承認済みの許可リストまたはレジストリに対して照合されていることを**検証します**。| 2 |
| **9.3.8** | ポリシー違反によってツールの自動隔離がトリガーされているることを**検証します**。| 3 |

## C9.4 エージェントおよびオーケストレーターのアイデンティティ

すべてのアクションは帰属可能であり、すべての変更は検出可能である必要があります。

| # | 説明 | レベル |
| :--: | --- | :---: |
| **9.4.1** | 各エージェント インスタンスが一意の暗号化アイデンティティを持ち、下流のシステムに対して第一級プリンシパルとして認証されていることを**検証します**。| 2 |
| **9.4.2** | エージェントが開始するアクションが、否認防止のために実行チェーンの各ステップに暗号化的にバインドされていることを**検証します**。| 2 |
| **9.4.3** | エージェント アイデンティティの資格情報が既定のスケジュールに従ってローテーションされることを**検証します**。| 3 |
| **9.4.4** | 呼び出し間で保持されるエージェントの状態が完全性保護されていることを**検証します**。| 3 |

---

## C9.5 エージェントの認可、委任、および継続的な適用

すべてのアクションは、実行時に承認され、スコープによって制約される必要があります。

| # | 説明 | レベル |
| :--: | --- | :---: |
| **9.5.1** | エージェントのアクションが、ランタイムによって適用されるきめ細かいポリシーに基づいて承認されていることを**検証します**。このポリシーは、エージェントが呼び出すことができるツールと、エージェントが提供できるパラメータ値を制限するものです。| 2 |
| **9.5.2** | エージェントがユーザーに代わってアクションを実行する場合、ランタイムは、ユーザーの認可コンテキストを保持し、完全性が保護されたスコープ限定トークンを伝播し、すべての下流の呼び出しで適用されていることを**検証します**。| 2 |
| **9.5.3** | すべてのアクセス制御の決定は、アプリケーション ロジックまたはポリシー エンジンによって適用され、AI モデル自体によって適用されていないことを**検証します**。| 2 |
| **9.5.4** | 実行時にエージェントが必要とするシークレットと資格情報が、コンテキスト ウィンドウ、システム プロンプト、ツール呼び出しのパラメータなど、モデルの観測可能なコンテキスト内に公開されていないことを**検証します**。| 2 |
| **9.5.5** | エージェント間のタスク委任が明示的な認可ポリシーによって制限されていることを**検証します**。| 2 |
| **9.5.6** | 長時間実行されるエージェント セッションが、特権アクションごとに現在のバックエンド認可ポリシーを再評価していることを**検証します**。| 3 |

---

## C9.6 シャットダウンとグレースフル デグラデーション

シャットダウンとグレースフル デグラデーションのパスは、信頼性が高く、定期的に実行される機構を備え、人間の制御下に置かれる必要があります。

| # | 説明 | レベル |
| :--: | --- | :---: |
| **9.6.1** | AI モデルの推論と出力を即座に停止する手動キルスイッチ機構が存在することを**検証します**。 | 1 |
| **9.6.2** | 既定の承認時間内に人間の承認ゲートが満たされない場合、システムが保留中のアクションをブロックしていることを**検証します**。 | 2 |
| **9.6.3** | キル スイッチ コマンドが、エージェント ランタイムから隔離された帯域外チャネルを介して実装されていることを**検証します**。 | 3 |
