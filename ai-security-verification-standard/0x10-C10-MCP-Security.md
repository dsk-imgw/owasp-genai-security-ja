# C10 モデル コンテキスト プロトコル (MCP) のセキュリティ

## 管理策目標

この章では、MCP ベースのツールおよびリソース統合における、セキュアな検出、認証、認可、転送、および使用について説明します。

---

## C10.1 コンポーネントの完全性

信頼できる MCP コンポーネントのみを使用し、ローカルで起動したサーバーはセキュリティが確保されている必要があります。

| # | 説明 | レベル |
| :--: | --- | :---: |
| **10.1.1** | MCP コンポーネントが信頼できるソースからのみ取得され、暗号化によって照合されていることを**検証します**。 | 1 |
| **10.1.2** | 許可リストに登録されている MCP サーバーのみが許可されていることを**検証します**。 | 2 |
| **10.1.3** | ローカルで起動された MCP サーバーが、ファイル システム、ネットワーク、およびシステムへのアクセスが制限された最小権限サンドボックスで実行されていることを**検証します**。 | 2 |

---

## C10.2 認証および認可

呼び出し元は、認証され、プロトコルのベストプラクティスに従って MCP サーバーへのアクセスが許可される必要があります。

| # | 説明 | レベル |
| :--: | --- | :---: |
| **10.2.1** | MCP サーバーは、各リクエストに対してアクセス トークンを確認 (validate) し、トランスポート セキュリティのみに依存していないことを**検証します**。| 1 |
| **10.2.2** | MCP サーバーは、提示されたアクセス トークンの発行者、対象者、有効期限、およびスコープ クレームを OAuth 2.1 に従って確認 (validate) していることを**検証します**。| 1 |
| **10.2.3** | OAuth 2.1 リソース サーバーとして機能する MCP サーバーは、アクセス トークンまたはユーザー資格情報を保存または永続化していないことを**検証します**。| 1 |
| **10.2.4** | MCP ツール/リストは、リソース所有者の承認済みスコープで許可されたツールのみを返していることを**検証します**。| 2 |
| **10.2.5** | MCP サーバーは、すべてのツール呼び出しに対してアクセス制御を適用し、ユーザーのアクセス トークンによって要求されたツールと指定された引数値の両方が承認されることを確認 (validate) していることを**検証します**。| 2 |
| **10.2.6** | MCP サーバーは、セッション終了時にすべてのセッション アーティファクトを削除していることを**検証します**。| 2 |
| **10.2.7** | MCP サーバーは、クライアントから受信したアクセス トークンを下流の API に渡していないことを**検証します**。| 2 |

---

## C10.3 セキュアな転送

MCP 通信は、プロトコルのベスト プラクティスに従ってセキュリティが確保されている必要があります。。

| # | 説明 | レベル |
| :--: | --- | :---: |
| **10.3.1** | 認証付きおよび暗号化された streamable HTTP が、リモート サービスへの MCP トランスポートに使用されていることを**検証します**。| 1 |
| **10.3.2** | stdio トランスポートは、管理されたローカル環境でのみ許可されていることを**検証します**。| 1 |
| **10.3.3** | DNS リバインディング攻撃を防ぐため、MCP サーバーはすべての HTTP ベースのトランスポートで Origin ヘッダーと Host ヘッダーの両方を個別に確認 (validate) していることを**検証します**。| 2 |
| **10.3.4** | MCP クライアントが許容可能な最小プロトコル バージョンを強制し、その最小バージョンを下回るバージョンを提案する初期化応答を拒否していることを**検証します**。| 2 |
| **10.3.5** | MCP クライアントと MCP サーバー間のアクセス トークンは、mTLS または DPoP を使用して送信者制約されていることを**検証します**。| 3 |

---

## C10.4 スキーマ、メッセージおよび入力の確認 (validation)

スキーマ、メッセージ、および入力の確認 (validation) は、MCP サーバーとクライアントの両方で実施される必要があります。

| # | 説明 | レベル |
| :--: | --- | :---: |
| **10.4.1** | MCP ツール/リストおよびツール/呼び出し応答は、モデル コンテキストに挿入される前に、宣言されたスキーマに対して確認 (validate) されていることを**検証します**。 | 1 |
| **10.4.2** | MCP ツール/リストおよびツール/呼び出し応答は、モデル コンテキストに挿入される前に、間接プロンプト インジェクションについてスクリーニングされていることを**検証します**。 | 1 |
| **10.4.3** | MCP サーバーは、関数呼び出しにおける認識されないパラメータまたは過大なパラメータを拒否していることを**検証します**。 | 1 |
| **10.4.4** | すべての MCP サーバーは、厳密なスキーマ確認 (validation) を強制していることを**検証します**。 | 2 |
| **10.4.5** | すべての MCP トランスポートは、最大ペイロード サイズ制限を強制していることを**検証します**。 | 2 |
| **10.4.6** | MCP サーバーは、ツール応答に固有のノンスとタイムスタンプを付与し、MCP クライアントがリプレイ攻撃を検出できるようになっていることを**検証します**。| 2 |
| **10.4.7** | MCP クライアントは、ローカル MCP サーバーのインストール時に、ユーザーに明示的な同意ダイアログとキャンセル オプションを表示していることを**検証します**。| 2 |
| **10.4.8** | MCP クライアントは、ツール定義のスナップショットを保持し、ツール定義に変更があった場合は、変更後のツールが呼び出される前に再承認が必要になっていることを**検証します**。| 3 |
