# C1 学習データの完全性および追跡可能性

## 管理策目標

この章では、学習データの取得、処理、および維持管理におけるデータの完全性と追跡可能性の保護について説明します。

---

## C1.1 学習データの出所とデータ セキュリティ

学習データの出所とセキュリティは、あらゆる AI システムの信頼性にとって極めて重要です。データセットは検証可能な出所から取得され、ライフサイクル全体にわたって追跡され、不正な変更を検出できるように、改ざん、破損、および汚染から保護されなければなりません。

| # | 説明 | レベル |
| :--------: | --------------------------------------------------------------------------------------------------------------------- | :---: |
| **1.1.1** | 学習データには、モデルの明示された目的に必要な特徴量、属性、およびフィールドのみが含まれていることを**検証します**。| 1 |
| **1.1.2** | すべての学習データ ソースについて、その出所、責任者、ライセンス、収集方法、使用目的の制約、処理履歴を含む最新のインベントリが保持されていることを**検証します**。 | 2 |
| **1.1.3** | 学習データの保存および転送時に、データの完全性が確保されていることを**検証します**。 | 2 |
| **1.1.4** | 学習データの不正な変更または破損を防ぐために、完全性監視が適用されていることを**検証します**。 | 2 |
| **1.1.5** | データセットに透かしが入れられており、使用状況を特定し不正使用を検出できることを**検証します**。 | 3 |

---

## C1.2 データ ラベリングおよびアノテーションのセキュリティ

ラベリングおよびアノテーションのプロセスは、不正な改ざん、データ漏洩、およびデータの完全性侵害から保護されなければなりません。アノテーション プラットフォームは、アクセス制御を徹底し、監査可能性を維持し、学習パイプライン全体を通してラベリング アーティファクトと機密性の高いラベル コンテンツを保護する必要があります。

| # | 説明 | レベル |
| :--------: | --------------------------------------------------------------------------------------------------------------------- | :---: |
| **1.2.1** | ラベリング プラットフォームが、アノテーションの作成、変更、承認を行うユーザーを制限するアクセス制御を適用していることを**検証します**。 | 1 |
| **1.2.2** | ラベリング アーティファクトに暗号化による完全性が適用されていることを**検証します**。 | 2 |
| **1.2.3** | ラベル内の機密情報が、ラベリング アーティファクトで使用される前に、秘匿化、匿名化、または暗号化されていることを**検証します**。 | 2 |

---

## C1.3 学習データの品質とセキュリティ保証

品質とセキュリティ保証のための管理策は、データ破損、汚染、ラベル エラー、悪用可能なデータセット パターンがモデルの動作に影響を与える前に検出するのに役立ちます。パイプラインは、自動検証、汚染検出、ラベル品質チェック、バイアス分析を組み合わせる必要があります。


| # | 説明 | レベル |
| :--------: | --------------------------------------------------------------------------------------------------------------------- | :---: |
| **1.3.1** | 学習および微調整パイプラインが、学習データにおける潜在的なデータ汚染または意図しない破損を特定するための汚染検出技術を実装していることを**検証します**。| 2 |
| **1.3.2** | 自動生成されたラベルが、誤解を招くラベルまたは信頼度の低いラベルを検出するために、信頼度閾値と一貫性チェックの対象となっていることを**検証します**。| 2 |
| **1.3.3** | セキュリティ関連の決定に使用されるモデルが、バイアス パターンについて評価されていることを**検証します**。| 2 |
| **1.3.4** | 学習前に、未許可のコンテンツが検出され、削除されていることを**検証します**。| 2 |
| **1.3.5** | クリーン ラベル汚染攻撃に対する防御策が実装されていることを**検証します**。| 3 |
