# C5 AI コンポーネントおよびユーザーのアクセス制御とアイデンティティ

## 管理策目標

この章では、AI システムが従来のアプリケーション セキュリティを超えてもたらすアクセス制御上の課題について説明します。

---

## C5.1 認証

リソースにアクセスする AI エージェントと人間のユーザーは、アクセス レベルに応じて適切に認証および認可される必要があります。

| # | 説明 | レベル |
| :--------: | --------------------------------------------------------------------------------------------- | :---: |
| **5.1.1** | 高リスクの AI 操作（モデルのデプロイ、重みのエクスポート、学習データへのアクセス、本番環境構成の変更）には、ステップアップ認証が必要とされていることを**検証します**。| 3 |
| **5.1.2** | フェデレーション環境またはマルチ システム環境における AI エージェントは、有効期限が短く、スコープが最小限で、暗号署名されたトークンを使用して認証されていることを**検証します**。| 3 |

---

## C5.2 AI リソースの認可と分類

呼び出し元の認可コンテキストは、AI 固有のクエリ パイプライン（RAG 取得、エンベディングのルックアップ、推論チェーン）を通じて強制的に適用され、呼び出し元がアクセス権限を持たないデータが返されないようにする必要があります。

| # | 説明 | レベル |
| :--------: | --------------------------------------------------------------------------------------------- | :---: |
| **5.2.1** | すべての AI リソース (データセット、エンドポイント、ベクトル コレクション、エンベディング インデックス、計算インスタンス) が、明示的な許可リストとデフォルト拒否ポリシーによるアクセス制御を適用していることを**検証します**。 | 2 |
| **5.2.2** | 取得パイプライン (RAG クエリ、エンベディングのルックアップなど) が、サービス アカウントの権限のみに依存するのではなく、各取得およびアセンブリ ステージでエンド ユーザーの認証コンテキストを適用していることを**検証します**。 | 2 |
| **5.2.3** | 機密データがモデルに永続的に保存されないように、取得パイプライン (RAG クエリ、エンベディングのルックアップなど) を介して機密データが取得されていることを**検証します**。 | 2 |
| **5.2.4** | 推論後のフィルタリング機構により、要求者が受信する権限のないデータが応答に含まれないようになっていることを**検証します**。| 2 |
| **5.2.5** | エージェント認可のポリシー決定ポイントが、エージェントの実行環境から分離されていることを**検証します**。| 2 |
| **5.2.6** | モデルの重み、学習パイプライン、および本番環境の AI 構成への特権アクセスが、既定の最大セッション期間と自動有効期限付きで、必要なときにのみ許可されていることを**検証します**。これらのリソースへのゼロ スタンディング特権（ZSP）が推奨されます。| 3 |
| **5.2.7** | データ分類ラベルが下流のリソース（エンベディング、プロンプト キャッシュ、モデル出力）に伝播していることを**検証します**。| 3 |

---

## C5.3 マルチ テナントの分離

推論キャッシュや共有モデル状態など、AI  固有の共有インフラストラクチャを介したテナント間の情報漏洩を防止する必要があります。

| # |説明 |レベル |
| :----------: | -------------------------------------------------------------------------------------------- | :---: |
| **5.3.1** | 共有モデル サービング インフラストラクチャによって、あるテナントの微調整（ファイン チューニング）、推論、エンベディング操作が別のテナントの操作に影響を与えたり、監視したりすることがないことを**検証します**。 | 2 |
| **5.3.2** | あるテナントが共有計算リソースを通じて別のテナントの操作に影響を与えたり、監視したりできないことを**検証します**。この要件を満たすには、通常、ハードウェアのパーティショニング、機密コンピューティング、またはテナントごとの専用コンピューティング割り当てが必要です。 | 3 |
